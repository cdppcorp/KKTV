## Is Your Code Really Safe?

Are you deploying AI-generated code without any review?

```python
# AI-generated code — do you know how many vulnerabilities are in here?
app.config['SECRET_KEY'] = 'super-secret-key-123'
query = f"SELECT * FROM users WHERE id = '{user_input}'"
app.run(debug=True)
```

The code above contains **3 security vulnerabilities**: a hardcoded secret key, SQL injection, and debug mode in production. Code generated by AI tools like Cursor, Claude Code, and Copilot ships with vulnerabilities like these every day.

**The problem is that most vibe coders don't even know this, and leave these vulnerabilities untouched.**

## If You Serve in Korea, KISA Guidelines Are Not Optional

The Korea Internet & Security Agency (KISA) published the **Python Secure Coding Guide (2023 Revision)**, which defines 47 security weaknesses and their countermeasures. If you operate a web service in Korea or participate in public sector projects, following these guidelines is **practically a legal obligation**.

- **Electronic Government Act, Article 45**: Mandatory application of software development security when building information systems for government agencies
- **Personal Information Protection Act (PIPA)**: Mandatory safety measures when processing personal information
- **Information and Communications Network Act**: Mandatory security vulnerability audits for information and communications service providers

> Even if you are a foreign developer, if you are entering the Korean market or working with Korean companies, you must be familiar with KISA guidelines.

## KKTV Solves This

KKTV is a plugin that turns the 47 rules from the KISA Secure Coding Guide into Claude Code skills. With a single command, it audits your project for guideline compliance and automatically fixes vulnerabilities.

### Before: Neglect and Exposure

- Deploying AI-generated code without any review
- Not even knowing security guidelines exist
- "I'll deal with it later" — then scrambling after an incident
- No security expert on the team, no idea where to start

### After: With KKTV

- One `/kktv.start` command audits all 47 rules
- Category-based reports show exactly where you're vulnerable
- `/kktv.fix` automatically patches discovered vulnerabilities
- `/kktv.guide` generates prompts that make AI write secure code from the start

## Quick Start

### 1. Installation

```bash
# Marketplace registration
/plugin marketplace add cdppcorp/KKTV

# English version
/plugin install kktv-en@kktv

# Korean version
/plugin install kktv-ko@kktv
```

### 2. Start Security Audit

```bash
/kktv.start
```

On first run, an environment questionnaire appears:
- Automatic OS detection (Windows/Linux)
- Automatic language and framework detection
- Deployment target, database, and authentication method confirmation
- Automatic plugin hook compatibility diagnosis on Windows

### 3. Review Reports and Fix

```
reports/security/
├── summary.md              ← Overview summary
├── cat1-input-validation.md ← Detailed by category
├── cat2-security-features.md
├── ...
└── cat7-api-misuse.md
```

When vulnerabilities are found:

```bash
/kktv.fix      # Auto-fix
/kktv.check    # Final pre-deployment check
```

## KISA 47 Security Rules

KKTV is based on the 47 security weaknesses from the KISA Python Secure Coding Guide (2023 Revision). All rules are mapped to the international standard CWE (Common Weakness Enumeration).

| Category | Rules | Key Items |
|----------|-------|-----------|
| Input Data Validation and Representation | 18 | SQL injection, XSS, CSRF, SSRF, code injection, command injection |
| Security Features | 16 | Authentication, authorization, encryption, hardcoded secrets, password policy |
| Time and State | 2 | TOCTOU race condition, infinite loop/recursion |
| Error Handling | 3 | Error message exposure, missing response, improper exception handling |
| Code Errors | 3 | Null dereference, resource release, deserialization |
| Encapsulation | 2 | Session data exposure, leftover debug code |
| API Misuse | 3 | DNS-based security decisions, vulnerable API usage |
