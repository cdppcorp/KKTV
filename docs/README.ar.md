## هل شفرتك البرمجية آمنة حقاً؟

هل تنشر الشفرة البرمجية التي أنشأها الذكاء الاصطناعي دون أي مراجعة؟

```python
# شفرة أنشأها الذكاء الاصطناعي — هل تعرف كم عدد الثغرات الموجودة هنا؟
app.config['SECRET_KEY'] = 'super-secret-key-123'
query = f"SELECT * FROM users WHERE id = '{user_input}'"
app.run(debug=True)
```

تحتوي الشفرة أعلاه على **3 ثغرات أمنية**: مفتاح سري مُضمَّن في الشفرة، حقن SQL، ووضع التصحيح في بيئة الإنتاج. الشفرة التي تُنشئها أدوات الذكاء الاصطناعي مثل Cursor وClaude Code وCopilot تُنشر يومياً بهذه الثغرات.

**المشكلة أن معظم مبرمجي الـ Vibe لا يعرفون هذا حتى، ويتركون هذه الثغرات دون معالجة.**

## إذا كنت تقدم خدمات في كوريا، فإرشادات KISA ليست اختيارية

نشرت وكالة الإنترنت والأمن الكورية (KISA) **دليل البرمجة الآمنة بلغة Python (نسخة 2023 المُعدَّلة)**، الذي يُحدد 47 نقطة ضعف أمنية وإجراءاتها المضادة. إذا كنت تُشغّل خدمة ويب في كوريا أو تشارك في مشاريع القطاع العام، فإن الالتزام بهذه الإرشادات هو **التزام قانوني عملياً**.

- **قانون الحكومة الإلكترونية، المادة 45**: تطبيق إلزامي لأمن تطوير البرمجيات عند بناء أنظمة المعلومات للجهات الحكومية
- **قانون حماية المعلومات الشخصية**: تدابير أمنية إلزامية عند معالجة المعلومات الشخصية
- **قانون شبكات المعلومات والاتصالات**: تدقيق إلزامي للثغرات الأمنية لمقدمي خدمات المعلومات والاتصالات

> حتى لو كنت مطوراً أجنبياً، إذا كنت تدخل السوق الكوري أو تعمل مع شركات كورية، يجب أن تكون على دراية بإرشادات KISA.

## KKTV يحل هذه المشكلة

KKTV هو إضافة تُحوِّل القواعد الـ 47 من دليل البرمجة الآمنة لـ KISA إلى مهارات Claude Code. بأمر واحد، يُدقق مشروعك للتحقق من الامتثال للإرشادات ويُصلح الثغرات تلقائياً.

### قبل: الإهمال والتعرض

- نشر الشفرة التي أنشأها الذكاء الاصطناعي دون أي مراجعة
- عدم معرفة أن إرشادات الأمان موجودة أصلاً
- "سأتعامل معها لاحقاً" — ثم الهلع بعد وقوع حادثة
- لا يوجد خبير أمني في الفريق، ولا تعرف من أين تبدأ

### بعد: مع KKTV

- أمر واحد `/kktv.start` يُدقق جميع القواعد الـ 47
- تقارير مُصنَّفة تُظهر بالضبط أين توجد الثغرات
- `/kktv.fix` يُصلح الثغرات المكتشفة تلقائياً
- `/kktv.guide` يُولِّد موجهات تجعل الذكاء الاصطناعي يكتب شفرة آمنة من البداية

## البدء السريع

### 1. التثبيت

```bash
# التسجيل في السوق
/plugin marketplace add cdppcorp/KKTV

# النسخة الإنجليزية
/plugin install kktv-en@kktv

# النسخة الكورية
/plugin install kktv-ko@kktv
```

### 2. بدء التدقيق الأمني

```bash
/kktv.start
```

عند التشغيل لأول مرة، يظهر استبيان البيئة:
- كشف تلقائي لنظام التشغيل (Windows/Linux)
- كشف تلقائي للغة البرمجة وإطار العمل
- تأكيد هدف النشر وقاعدة البيانات وطريقة المصادقة
- تشخيص تلقائي لتوافق خطافات الإضافة على Windows

### 3. مراجعة التقارير والإصلاح

```
reports/security/
├── summary.md              ← ملخص عام
├── cat1-input-validation.md ← تفاصيل حسب الفئة
├── cat2-security-features.md
├── ...
└── cat7-api-misuse.md
```

عند اكتشاف ثغرات:

```bash
/kktv.fix      # إصلاح تلقائي
/kktv.check    # فحص نهائي قبل النشر
```

## قواعد الأمان الـ 47 من KISA

يستند KKTV إلى 47 نقطة ضعف أمنية من دليل البرمجة الآمنة بلغة Python لـ KISA (نسخة 2023 المُعدَّلة). جميع القواعد مُعيَّنة وفق المعيار الدولي CWE (Common Weakness Enumeration).

| الفئة | عدد القواعد | البنود الرئيسية |
|-------|-------------|-----------------|
| التحقق من بيانات الإدخال والتمثيل | 18 | حقن SQL، XSS، CSRF، SSRF، حقن الشفرة، حقن الأوامر |
| ميزات الأمان | 16 | المصادقة، التفويض، التشفير، الأسرار المُضمَّنة، سياسة كلمات المرور |
| الوقت والحالة | 2 | حالة سباق TOCTOU، حلقة لا نهائية/تكرار |
| معالجة الأخطاء | 3 | كشف رسائل الخطأ، غياب الاستجابة، معالجة استثناءات غير مناسبة |
| أخطاء الشفرة | 3 | إلغاء مرجعية Null، تحرير الموارد، إلغاء التسلسل |
| التغليف | 2 | كشف بيانات الجلسة، بقاء شفرة التصحيح |
| سوء استخدام API | 3 | قرارات أمنية مبنية على DNS، استخدام واجهات برمجية ضعيفة |
