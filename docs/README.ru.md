## Ваш код действительно безопасен?

Вы разворачиваете код, сгенерированный ИИ, без какой-либо проверки?

```python
# Код, сгенерированный ИИ — вы знаете, сколько здесь уязвимостей?
app.config['SECRET_KEY'] = 'super-secret-key-123'
query = f"SELECT * FROM users WHERE id = '{user_input}'"
app.run(debug=True)
```

В приведённом коде **3 уязвимости безопасности**: жёстко закодированный секретный ключ, SQL-инъекция и режим отладки в продакшене. Код, сгенерированный ИИ-инструментами вроде Cursor, Claude Code и Copilot, ежедневно разворачивается с подобными уязвимостями.

**Проблема в том, что большинство вайб-кодеров даже не знают об этом и оставляют эти уязвимости без внимания.**

## Если вы работаете на корейском рынке, рекомендации KISA обязательны

Корейское агентство по интернету и безопасности (KISA) опубликовало **Руководство по безопасному кодированию на Python (редакция 2023 года)**, которое определяет 47 уязвимостей безопасности и меры противодействия. Если вы управляете веб-сервисом в Корее или участвуете в проектах государственного сектора, соблюдение этих рекомендаций — **практически юридическая обязанность**.

- **Закон об электронном правительстве, статья 45**: обязательное применение безопасности при разработке ПО для информационных систем государственных органов
- **Закон о защите персональных данных**: обязательные меры безопасности при обработке персональных данных
- **Закон об информационно-коммуникационных сетях**: обязательный аудит уязвимостей безопасности для провайдеров информационно-коммуникационных услуг

> Даже если вы иностранный разработчик, при выходе на корейский рынок или сотрудничестве с корейскими компаниями вы должны знать рекомендации KISA.

## KKTV решает эту проблему

KKTV — это плагин, который превращает 47 правил из руководства KISA по безопасному кодированию в навыки Claude Code. Одной командой он проверяет ваш проект на соответствие рекомендациям и автоматически исправляет уязвимости.

### До: Халатность и уязвимость

- Развёртывание кода, сгенерированного ИИ, без какой-либо проверки
- Незнание о существовании рекомендаций по безопасности
- «Займусь этим позже» — а потом аврал после инцидента
- Нет специалиста по безопасности, непонятно, с чего начать

### После: С KKTV

- Одна команда `/kktv.start` проверяет все 47 правил
- Отчёты по категориям сразу показывают, где уязвимости
- `/kktv.fix` автоматически исправляет обнаруженные уязвимости
- `/kktv.guide` генерирует промпты, чтобы ИИ писал безопасный код с самого начала

## Быстрый старт

### 1. Установка

```bash
# Регистрация в маркетплейсе
/plugin marketplace add cdppcorp/KKTV

# Английская версия
/plugin install kktv-en@kktv

# Корейская версия
/plugin install kktv-ko@kktv
```

### 2. Запуск аудита безопасности

```bash
/kktv.start
```

При первом запуске появляется анкета окружения:
- Автоматическое определение ОС (Windows/Linux)
- Автоматическое определение языка и фреймворка
- Уточнение цели развёртывания, БД и метода аутентификации
- Автоматическая диагностика совместимости хуков плагина в Windows

### 3. Просмотр отчётов и исправление

```
reports/security/
├── summary.md              ← Обзорная сводка
├── cat1-input-validation.md ← Детализация по категориям
├── cat2-security-features.md
├── ...
└── cat7-api-misuse.md
```

При обнаружении уязвимостей:

```bash
/kktv.fix      # Автоисправление
/kktv.check    # Финальная проверка перед развёртыванием
```

## 47 правил безопасности KISA

KKTV основан на 47 уязвимостях безопасности из руководства KISA по безопасному кодированию на Python (редакция 2023 года). Все правила сопоставлены с международным стандартом CWE (Common Weakness Enumeration).

| Категория | Правил | Основные пункты |
|-----------|--------|-----------------|
| Валидация входных данных и представление | 18 | SQL-инъекция, XSS, CSRF, SSRF, инъекция кода, инъекция команд |
| Функции безопасности | 16 | Аутентификация, авторизация, шифрование, жёстко закодированные секреты, политика паролей |
| Время и состояние | 2 | Состояние гонки TOCTOU, бесконечный цикл/рекурсия |
| Обработка ошибок | 3 | Раскрытие информации в сообщениях об ошибках, отсутствие обработки, некорректная обработка исключений |
| Ошибки в коде | 3 | Разыменование null, освобождение ресурсов, десериализация |
| Инкапсуляция | 2 | Раскрытие данных сессии, оставшийся отладочный код |
| Неправильное использование API | 3 | Решения безопасности на основе DNS, использование уязвимых API |
