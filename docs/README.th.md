## โค้ดของคุณปลอดภัยจริงหรือ?

คุณกำลัง deploy โค้ดที่ AI สร้างขึ้นโดยไม่ผ่านการตรวจสอบใดเลยหรือเปล่า?

```python
# โค้ดที่ AI สร้าง — คุณรู้ไหมว่ามีช่องโหว่กี่จุดในนี้?
app.config['SECRET_KEY'] = 'super-secret-key-123'
query = f"SELECT * FROM users WHERE id = '{user_input}'"
app.run(debug=True)
```

โค้ดด้านบนมี **ช่องโหว่ด้านความปลอดภัย 3 จุด**: secret key ที่ฝังในโค้ด, SQL injection และ debug mode ใน production โค้ดที่สร้างโดยเครื่องมือ AI เช่น Cursor, Claude Code และ Copilot ถูก deploy ทุกวันพร้อมกับช่องโหว่เหล่านี้

**ปัญหาคือ vibe coder ส่วนใหญ่ไม่รู้เรื่องนี้เลย และปล่อยให้ช่องโหว่เหล่านี้คงอยู่**

## หากคุณให้บริการในเกาหลี แนวทาง KISA ไม่ใช่ทางเลือก

สำนักงานอินเทอร์เน็ตและความปลอดภัยแห่งเกาหลี (KISA) ได้เผยแพร่ **คู่มือการเขียนโค้ดปลอดภัย Python (ฉบับปรับปรุง 2023)** ซึ่งกำหนดจุดอ่อนด้านความปลอดภัย 47 ข้อและมาตรการรับมือ หากคุณดำเนินบริการเว็บในเกาหลีหรือเข้าร่วมโครงการภาครัฐ การปฏิบัติตามแนวทางเหล่านี้ถือเป็น **ข้อบังคับทางกฎหมายโดยพฤตินัย**

- **พระราชบัญญัติรัฐบาลอิเล็กทรอนิกส์ มาตรา 45**: บังคับใช้ความปลอดภัยในการพัฒนาซอฟต์แวร์สำหรับระบบสารสนเทศของหน่วยงานราชการ
- **พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล**: บังคับใช้มาตรการความปลอดภัยในการประมวลผลข้อมูลส่วนบุคคล
- **พระราชบัญญัติเครือข่ายสารสนเทศและการสื่อสาร**: บังคับตรวจสอบช่องโหว่ด้านความปลอดภัยสำหรับผู้ให้บริการสารสนเทศและการสื่อสาร

> แม้คุณจะเป็นนักพัฒนาต่างชาติ หากคุณกำลังเข้าสู่ตลาดเกาหลีหรือทำงานร่วมกับบริษัทเกาหลี คุณต้องรู้จักแนวทาง KISA

## KKTV แก้ปัญหานี้

KKTV เป็นปลั๊กอินที่แปลง 47 กฎจากคู่มือการเขียนโค้ดปลอดภัยของ KISA ให้เป็นทักษะ Claude Code เพียงคำสั่งเดียว ตรวจสอบโปรเจกต์ของคุณว่าสอดคล้องกับแนวทางหรือไม่ และแก้ไขช่องโหว่โดยอัตโนมัติ

### ก่อน: ละเลยและเปิดเผย

- Deploy โค้ดที่ AI สร้างโดยไม่ตรวจสอบใดเลย
- ไม่รู้ด้วยซ้ำว่ามีแนวทางด้านความปลอดภัยอยู่
- "ไว้ทำทีหลัง" — แล้วก็วุ่นวายหลังเกิดเหตุ
- ไม่มีผู้เชี่ยวชาญด้านความปลอดภัยในทีม ไม่รู้จะเริ่มจากตรงไหน

### หลัง: ใช้ KKTV

- คำสั่ง `/kktv.start` เพียงครั้งเดียว ตรวจสอบทั้ง 47 กฎ
- รายงานตามหมวดหมู่แสดงให้เห็นชัดเจนว่าจุดไหนมีช่องโหว่
- `/kktv.fix` แก้ไขช่องโหว่ที่พบโดยอัตโนมัติ
- `/kktv.guide` สร้าง prompt ให้ AI เขียนโค้ดปลอดภัยตั้งแต่แรก

## เริ่มต้นอย่างรวดเร็ว

### 1. ติดตั้ง

```bash
# ลงทะเบียน marketplace
/plugin marketplace add cdppcorp/KKTV

# เวอร์ชันภาษาอังกฤษ
/plugin install kktv-en@kktv

# เวอร์ชันภาษาเกาหลี
/plugin install kktv-ko@kktv
```

### 2. เริ่มการตรวจสอบความปลอดภัย

```bash
/kktv.start
```

เมื่อรันครั้งแรก แบบสอบถามสภาพแวดล้อมจะปรากฏ:
- ตรวจจับ OS อัตโนมัติ (Windows/Linux)
- ตรวจจับภาษาและ framework อัตโนมัติ
- ยืนยันเป้าหมาย deploy, ฐานข้อมูล และวิธีการยืนยันตัวตน
- วินิจฉัยความเข้ากันได้ของ plugin hook บน Windows อัตโนมัติ

### 3. ดูรายงานและแก้ไข

```
reports/security/
├── summary.md              ← สรุปภาพรวม
├── cat1-input-validation.md ← รายละเอียดตามหมวดหมู่
├── cat2-security-features.md
├── ...
└── cat7-api-misuse.md
```

เมื่อพบช่องโหว่:

```bash
/kktv.fix      # แก้ไขอัตโนมัติ
/kktv.check    # ตรวจสอบครั้งสุดท้ายก่อน deploy
```

## 47 กฎความปลอดภัย KISA

KKTV อ้างอิงจากจุดอ่อนด้านความปลอดภัย 47 ข้อจากคู่มือการเขียนโค้ดปลอดภัย Python ของ KISA (ฉบับปรับปรุง 2023) กฎทั้งหมดถูก map กับมาตรฐานสากล CWE (Common Weakness Enumeration)

| หมวดหมู่ | จำนวนกฎ | รายการหลัก |
|----------|---------|------------|
| การตรวจสอบข้อมูลนำเข้าและการแสดงผล | 18 | SQL injection, XSS, CSRF, SSRF, code injection, command injection |
| ฟีเจอร์ด้านความปลอดภัย | 16 | การยืนยันตัวตน, การอนุญาต, การเข้ารหัส, secret ที่ฝังในโค้ด, นโยบายรหัสผ่าน |
| เวลาและสถานะ | 2 | TOCTOU race condition, infinite loop/recursion |
| การจัดการข้อผิดพลาด | 3 | การเปิดเผยข้อความผิดพลาด, ไม่มีการตอบสนอง, การจัดการ exception ที่ไม่เหมาะสม |
| ข้อผิดพลาดของโค้ด | 3 | Null dereference, การปล่อยทรัพยากร, deserialization |
| การห่อหุ้ม | 2 | การเปิดเผยข้อมูล session, โค้ด debug ที่ตกค้าง |
| การใช้ API ผิดวิธี | 3 | การตัดสินด้านความปลอดภัยบน DNS, การใช้ API ที่มีช่องโหว่ |
