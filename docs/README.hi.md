## क्या आपका कोड सच में सुरक्षित है?

क्या आप AI द्वारा जनरेट किया गया कोड बिना किसी समीक्षा के डिप्लॉय कर रहे हैं?

```python
# AI द्वारा जनरेट किया गया कोड — क्या आप जानते हैं इसमें कितनी कमजोरियाँ हैं?
app.config['SECRET_KEY'] = 'super-secret-key-123'
query = f"SELECT * FROM users WHERE id = '{user_input}'"
app.run(debug=True)
```

ऊपर दिए गए कोड में **3 सुरक्षा कमजोरियाँ** हैं: हार्डकोडेड सीक्रेट की, SQL इंजेक्शन, और प्रोडक्शन में डिबग मोड। Cursor, Claude Code, और Copilot जैसे AI टूल्स द्वारा जनरेट किया गया कोड हर दिन इन्हीं कमजोरियों के साथ डिप्लॉय होता है।

**समस्या यह है कि अधिकांश वाइब कोडर्स को इसकी जानकारी ही नहीं होती और वे इन कमजोरियों को अनदेखा कर देते हैं।**

---

## अगर आप कोरिया में सेवा देते हैं, तो KISA गाइडलाइन वैकल्पिक नहीं हैं

कोरिया इंटरनेट एवं सुरक्षा एजेंसी (KISA) ने **Python सिक्योर कोडिंग गाइड (2023 संशोधित संस्करण)** प्रकाशित किया है, जो 47 सुरक्षा कमजोरियों और उनके उपायों को परिभाषित करता है। अगर आप कोरिया में वेब सेवा संचालित करते हैं या सार्वजनिक क्षेत्र की परियोजनाओं में भाग लेते हैं, तो इन गाइडलाइन का पालन करना **व्यावहारिक रूप से कानूनी बाध्यता है**।

- **इलेक्ट्रॉनिक गवर्नमेंट एक्ट, धारा 45**: सरकारी एजेंसियों के सूचना प्रणालियों के लिए सॉफ्टवेयर विकास सुरक्षा का अनिवार्य अनुप्रयोग
- **व्यक्तिगत सूचना संरक्षण अधिनियम**: व्यक्तिगत जानकारी प्रसंस्करण के दौरान अनिवार्य सुरक्षा उपाय
- **सूचना एवं संचार नेटवर्क अधिनियम**: सूचना एवं संचार सेवा प्रदाताओं के लिए अनिवार्य सुरक्षा कमजोरी ऑडिट

> भले ही आप विदेशी डेवलपर हों, अगर आप कोरियाई बाज़ार में प्रवेश कर रहे हैं या कोरियाई कंपनियों के साथ काम कर रहे हैं, तो आपको KISA गाइडलाइन से परिचित होना चाहिए।

## KKTV इसे हल करता है

KKTV एक प्लगइन है जो KISA सिक्योर कोडिंग गाइड के 47 नियमों को Claude Code स्किल्स में बदलता है। एक ही कमांड से, यह आपकी परियोजना का गाइडलाइन अनुपालन ऑडिट करता है और स्वचालित रूप से कमजोरियों को ठीक करता है।

### पहले: उपेक्षा और जोखिम

- AI द्वारा जनरेट कोड को बिना किसी समीक्षा के डिप्लॉय करना
- सुरक्षा गाइडलाइन के अस्तित्व के बारे में जानकारी न होना
- "बाद में कर लूँगा" — फिर घटना के बाद भागमभाग
- टीम में सुरक्षा विशेषज्ञ नहीं, कहाँ से शुरू करें पता नहीं

### बाद में: KKTV के साथ

- एक `/kktv.start` कमांड से सभी 47 नियमों का ऑडिट
- श्रेणी-आधारित रिपोर्ट तुरंत दिखाती हैं कि कमजोरियाँ कहाँ हैं
- `/kktv.fix` खोजी गई कमजोरियों को स्वचालित रूप से ठीक करता है
- `/kktv.guide` प्रॉम्प्ट जनरेट करता है ताकि AI शुरू से ही सुरक्षित कोड लिखे

## त्वरित शुरुआत

### 1. इंस्टॉलेशन

```bash
# मार्केटप्लेस रजिस्ट्रेशन
/plugin marketplace add cdppcorp/KKTV

# अंग्रेजी संस्करण
/plugin install kktv-en@kktv

# कोरियाई संस्करण
/plugin install kktv-ko@kktv
```

### 2. सुरक्षा ऑडिट शुरू करें

```bash
/kktv.start
```

पहली बार चलाने पर, पर्यावरण प्रश्नावली दिखाई देती है:
- स्वचालित OS पहचान (Windows/Linux)
- स्वचालित भाषा और फ्रेमवर्क पहचान
- डिप्लॉयमेंट लक्ष्य, डेटाबेस, और प्रमाणीकरण विधि की पुष्टि
- Windows पर प्लगइन हुक संगतता का स्वचालित निदान

### 3. रिपोर्ट देखें और ठीक करें

```
reports/security/
├── summary.md              ← सारांश अवलोकन
├── cat1-input-validation.md ← श्रेणी-वार विवरण
├── cat2-security-features.md
├── ...
└── cat7-api-misuse.md
```

जब कमजोरियाँ मिलती हैं:

```bash
/kktv.fix      # स्वचालित सुधार
/kktv.check    # डिप्लॉयमेंट से पहले अंतिम जाँच
```

## KISA 47 सुरक्षा नियम

KKTV, KISA Python सिक्योर कोडिंग गाइड (2023 संशोधित संस्करण) की 47 सुरक्षा कमजोरियों पर आधारित है। सभी नियम अंतर्राष्ट्रीय मानक CWE (Common Weakness Enumeration) से मैप किए गए हैं।

| श्रेणी | नियम संख्या | प्रमुख आइटम |
|--------|-------------|-------------|
| इनपुट डेटा सत्यापन और प्रतिनिधित्व | 18 | SQL इंजेक्शन, XSS, CSRF, SSRF, कोड इंजेक्शन, कमांड इंजेक्शन |
| सुरक्षा सुविधाएँ | 16 | प्रमाणीकरण, प्राधिकरण, एन्क्रिप्शन, हार्डकोडेड सीक्रेट, पासवर्ड नीति |
| समय और स्थिति | 2 | TOCTOU रेस कंडीशन, अनंत लूप/रिकर्शन |
| त्रुटि प्रबंधन | 3 | त्रुटि संदेश एक्सपोज़र, प्रतिक्रिया की कमी, अनुचित अपवाद हैंडलिंग |
| कोड त्रुटियाँ | 3 | Null डीरेफरेंस, संसाधन रिलीज़, डीसीरियलाइज़ेशन |
| एनकैप्सुलेशन | 2 | सेशन डेटा एक्सपोज़र, बचा हुआ डिबग कोड |
| API दुरुपयोग | 3 | DNS-आधारित सुरक्षा निर्णय, कमजोर API उपयोग |
