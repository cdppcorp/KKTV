# Glossary — 용어 사전

> 집필 중 사용되는 주요 용어를 정리합니다. 새로운 용어가 등장하면 추가하세요.

| 한글 | 영문 | 약어 | 설명 |
|------|------|------|------|
| 바이브 코딩 | Vibe Coding | - | AI 도구를 활용하여 자연어로 코드를 생성하는 개발 방식 |
| SQL 삽입 | SQL Injection | SQLi | 악의적 SQL 구문을 입력하여 데이터베이스를 조작하는 공격 |
| 크로스사이트 스크립트 | Cross-Site Scripting | XSS | 웹 페이지에 악성 스크립트를 삽입하는 공격 |
| 크로스사이트 요청 위조 | Cross-Site Request Forgery | CSRF | 사용자의 인증된 세션을 이용하여 의도하지 않은 요청을 보내는 공격 |
| 서버사이드 요청 위조 | Server-Side Request Forgery | SSRF | 서버가 공격자가 지정한 URL로 요청을 보내도록 유도하는 공격 |
| 운영체제 명령어 삽입 | OS Command Injection | - | 시스템 명령어를 주입하여 서버를 조작하는 공격 |
| 경로 조작 | Path Traversal | - | 파일 경로를 조작하여 허가되지 않은 파일에 접근하는 공격 |
| 암호화 | Encryption | - | 데이터를 읽을 수 없는 형태로 변환하는 보안 기법 |
| 인증 | Authentication | AuthN | 사용자의 신원을 확인하는 절차 |
| 인가 | Authorization | AuthZ | 인증된 사용자에게 자원 접근 권한을 부여하는 절차 |
| 역직렬화 | Deserialization | - | 직렬화된 데이터를 객체로 복원하는 과정 |
| 하드코딩 | Hardcoding | - | 소스 코드에 민감 정보를 직접 기입하는 행위 |
| 시큐어코딩 | Secure Coding | - | 보안 취약점을 사전에 방지하는 코딩 방법론 |
| 입력값 검증 | Input Validation | - | 사용자 입력이 안전한지 확인하는 절차 |
| 에러 처리 | Error Handling | - | 오류 발생 시 안전하게 대응하는 코드 패턴 |
