# Constitution — 시큐어코딩 가이드라인 Skills

> 이 문서는 집필의 기본 규칙을 정의합니다. 수정이 필요하면 직접 편집하세요.

## 1. 문체 규칙

### 문장 종결 어미
- **합쇼체** 사용 (예: ~합니다, ~됩니다, ~하십시오)
- 격식체 유지

### 경어 수준
- 격식체 사용
- 독자를 "여러분" 또는 "사용자"로 지칭

### 예외
- 코드 주석: 해라체 허용 (예: "값을 검증한다")
- 팁/주의 박스 내부: 해요체 허용 (예: "주의하세요")

## 2. 특수 요소

### 팁 박스
```
> **💡 팁:** 내용
```

### 주의 박스
```
> **⚠️ 주의:** 내용
```

### 코드 예제
- 나쁜 예(취약 코드)와 좋은 예(안전 코드)를 반드시 함께 제시
- 나쁜 예 헤더: `❌ 취약한 코드`
- 좋은 예 헤더: `✅ 안전한 코드`
- 모든 코드 블록에 언어 태그 명시 (```python, ```html 등)

## 3. 용어 표기
- 용어 표기 형식: 자유 (저자 재량)
- 최초 등장 시 가능한 한 영문 병기 권장 (예: SQL 삽입(SQL Injection))
- 이후 등장 시 한글 또는 영문 중 가독성이 높은 쪽 사용

## 4. 상호참조
- 같은 문서 내: "N장 M절 참고" 또는 "위의 OO 참고"
- 외부 참조: URL 링크 사용

## 5. 대상 독자 고려
- **바이브 코딩**을 하는 사람, 특히 웹사이트를 바이브 코딩해서 퍼블리싱하는 사람이 대상
- AI 코드 생성 도구(Cursor, Claude Code, Copilot 등)를 사용하는 맥락을 전제
- 전통적 개발자가 아닌 독자를 위해 보안 개념을 쉽게 설명
- 실무 적용 가능한 체크리스트와 즉시 사용 가능한 코드 스니펫 중심

## 6. 장/절 구성 원칙
- 각 취약점 항목은 독립적으로 읽을 수 있어야 함
- 항목별 구조: 개요 → 왜 위험한가 → 취약 코드 → 안전 코드 → 바이브 코딩 시 체크포인트
- 실제 바이브 코딩 시나리오 기반 예제 우선
